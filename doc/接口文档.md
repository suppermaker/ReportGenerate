# 报告生成平台接口文档

版本：v2.1  
状态：设计稿  
适用服务：ReportGenerate 后端服务  
基础路径：`/api`

## 1. 文档说明

本文档用于前后端联调、后端开发、接口测试和验收。

当前代码中已落地的接口较少，本文档覆盖目标接口设计。接口字段以后端领域模型和任务表规划为准，实际实现时如涉及数据库结构、状态机或外部接口变更，应同步更新本文档。

接口状态说明：

| 状态 | 说明 |
| --- | --- |
| 已实现 | 当前代码已提供或可直接联调 |
| 规划中 | 需求和架构设计要求，需要后续实现 |
| 管理端 | 面向平台管理员、审计管理员或运维人员 |
| 内部 | 主要由后端模块、Worker 或系统任务调用 |

## 2. 通用约定

### 2.1 请求协议

- 协议：HTTP / HTTPS；任务进度推送支持 WebSocket 或 Server-Sent Events。
- 数据格式：JSON。
- 字符集：UTF-8。
- 文件上传：`multipart/form-data`。
- 默认服务地址：`http://localhost:8080`。

### 2.2 通用请求头

| Header | 必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `Authorization` | 是 | `Bearer eyJ...` | 登录后获取的 JWT；公开接口除外 |
| `Content-Type` | 是 | `application/json` | JSON 接口使用 |
| `X-Request-Id` | 否 | `REQ202606300001` | 请求链路 ID；不传由后端生成 |
| `Idempotency-Key` | 部分接口 | `submit-20260630-0001` | 幂等键，提交生成、重试、重新生成等写接口建议必传 |

### 2.3 统一响应结构

所有业务接口统一返回 `ApiResponse`。

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "REQ202606300001"
}
```

分页响应统一放在 `data` 中。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "total": 0,
    "pageNum": 1,
    "pageSize": 10
  },
  "requestId": "REQ202606300001"
}
```

### 2.4 通用错误码

| code | HTTP 状态码 | 含义 | 说明 |
| --- | --- | --- | --- |
| `0` | 200 | 成功 | 请求处理成功 |
| `40000` | 400 | 参数错误 | 参数缺失、格式错误、校验失败 |
| `40100` | 401 | 未认证 | 未登录或 Token 无效 |
| `40300` | 403 | 无权限 | 当前用户无权访问资源 |
| `40400` | 404 | 资源不存在 | 数据不存在或已删除 |
| `40900` | 409 | 状态冲突 | 当前状态不允许该操作 |
| `42900` | 429 | 请求过频 | 触发限流或重复提交 |
| `50000` | 500 | 系统异常 | 未预期的服务端异常 |
| `50200` | 502 | 外部服务异常 | Agent、知识库、MinIO 等外部服务异常 |
| `50300` | 503 | 服务不可用 | 依赖服务不可用或系统维护 |

## 3. 枚举定义

### 3.1 报告类型 `reportType`

| 值 | 说明 |
| --- | --- |
| `CRYPTO_SECURITY_ASSESSMENT` | 密码安全应用测评报告 |
| `WATER_CONSERVANCY` | 水利行业文档 |
| `CUSTOM` | 自定义报告 |

### 3.2 报告状态 `reportStatus`

| 值 | 说明 |
| --- | --- |
| `DRAFT` | 草稿 |
| `SUBMITTED` | 已提交 |
| `GENERATING` | 生成中 |
| `KNOWLEDGE_RETRIEVING` | 知识库检索中 |
| `AGENT_PROCESSING` | Agent 处理中 |
| `RENDERING` | 渲染中 |
| `REVIEWING` | 待人工复核 |
| `COMPLETED` | 已完成 |
| `FAILED` | 生成失败 |
| `RETRYING` | 重试中 |
| `ARCHIVED` | 已归档 |
| `DELETED` | 已删除 |

### 3.3 任务状态 `taskStatus`

| 值 | 说明 |
| --- | --- |
| `PENDING` | 待执行 |
| `QUEUED` | 已入队 |
| `RUNNING` | 执行中 |
| `SUCCESS` | 执行成功 |
| `FAILED` | 执行失败 |
| `RETRYING` | 重试中 |
| `CANCELED` | 已取消 |

说明：`taskStatus=SUCCESS` 表示生成任务执行成功；`reportStatus=COMPLETED` 表示报告业务生命周期已完成。

### 3.4 任务阶段 `stageCode`

| 值 | 说明 |
| --- | --- |
| `INIT` | 初始化 |
| `FILE_PREPARING` | 文件准备 |
| `DOCUMENT_PARSING` | 文档解析 |
| `KNOWLEDGE_RETRIEVAL` | 知识库检索 |
| `AGENT_GENERATION` | Agent 生成 |
| `COMPLIANCE_REVIEW` | 合规审查 |
| `RENDERING` | 渲染导出 |
| `UPLOAD_RESULT` | 上传生成结果 |
| `FINISH` | 完成 |

### 3.5 文件业务类型 `bizType`

| 值 | 说明 |
| --- | --- |
| `TEMPLATE` | 模板文件 |
| `REFERENCE` | 参考文档 |
| `REPORT` | 生成报告 |
| `PREVIEW` | 预览文件 |
| `OTHER` | 其他文件 |

### 3.6 模板状态 `templateStatus`

| 值 | 说明 |
| --- | --- |
| `DRAFT` | 草稿 |
| `ENABLED` | 启用 |
| `DISABLED` | 停用 |

### 3.7 导出格式 `exportFormat`

| 值 | 说明 |
| --- | --- |
| `PDF` | PDF 文件 |
| `DOCX` | Word 文件 |
| `MARKDOWN` | Markdown 文件 |
| `HTML` | HTML 预览文件 |

## 6. 文件管理接口

### 6.1 上传文件

```http
POST /api/files
Content-Type: multipart/form-data
```

状态：规划中

说明：上传模板、参考文档、生成报告或预览文件到 MinIO，并写入 `file_object` 元数据。文件不直接暴露 MinIO 地址。

权限：登录用户

表单参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `file` | file | 是 | 上传文件 |
| `bizType` | string | 是 | 文件业务类型，见 `bizType` |
| `bizId` | long | 否 | 关联业务 ID，上传时未知可为空 |
| `objectName` | string | 否 | 指定对象路径；不传由后端生成 |

响应数据：

```json
{
  "id": 10001,
  "fileCode": "FILE202606300001",
  "bizType": "REFERENCE",
  "bizId": 20001,
  "originalFilename": "测评材料.pdf",
  "objectName": "/references/1/20001/10001/测评材料.pdf",
  "bucketName": "report-generate",
  "contentType": "application/pdf",
  "fileExt": "pdf",
  "fileSize": 204800,
  "fileHash": "sha256:xxxx",
  "status": "ACTIVE"
}
```

### 6.2 查询文件详情

```http
GET /api/files/{fileId}
```

状态：规划中

说明：按文件 ID 查询文件元数据。

权限：登录用户，需具备文件所属业务资源访问权限。

### 6.3 获取文件临时访问 URL

```http
GET /api/files/{fileId}/access-url
```

状态：规划中

说明：后端完成权限校验后生成预签名 URL，用于预览或下载。

权限：登录用户，需具备文件访问权限。

查询参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `usage` | string | 否 | `DOWNLOAD` | `DOWNLOAD` 或 `PREVIEW` |
| `expireSeconds` | integer | 否 | `600` | URL 有效期，建议不超过 3600 秒 |

### 6.4 逻辑删除文件

```http
DELETE /api/files/{fileId}
```

状态：规划中

说明：逻辑删除文件元数据。物理文件可由定时清理任务处理。

权限：文件所有者或管理员。

## 16. 权限控制

### 16.1 角色定义

| 角色 | 典型权限 |
| --- | --- |
| `PLATFORM_ADMIN` | 全部功能、用户权限、模板管理、系统配置、所有报告 |
| `TEMPLATE_ADMIN` | 模板上传、下载、启停、变量维护 |
| `REPORT_CREATOR` | 创建项目、上传参考文档、提交生成、查看和管理自己创建的报告 |
| `REPORT_VIEWER` | 查看和下载被授权报告 |
| `AUDIT_ADMIN` | 查看审计日志、Agent 调用日志、知识库检索日志 |

### 16.2 接口级权限矩阵

| 模块 | 接口范围 | 主要角色 |
| --- | --- | --- |
| 认证 | `/api/auth/**` | 公开接口或登录用户 |
| 用户权限 | `/api/users/**`、`/api/roles/**`、`/api/permissions/**` | `PLATFORM_ADMIN` |
| 文件 | `/api/files/**` | 文件所有者、业务资源访问者、`PLATFORM_ADMIN` |
| 模板 | `/api/templates/**` | 查询为登录用户；写操作为 `TEMPLATE_ADMIN`、`PLATFORM_ADMIN` |
| 项目 | `/api/projects/**` | 项目创建者、被授权用户、`PLATFORM_ADMIN` |
| 任务 | `/api/tasks/**`、`/api/ws/tasks/**` | 任务所属报告访问者、`PLATFORM_ADMIN` |
| 报告 | `/api/reports/**` | 报告创建者、被授权用户、`REPORT_VIEWER`、`PLATFORM_ADMIN` |
| 分享 | `/api/share/**` | 公开访问，但必须校验分享有效期、密码和下载权限 |
| Agent | `/api/agents/**` | 查询推荐为登录用户；配置、调试、日志为 `PLATFORM_ADMIN` 或 `AUDIT_ADMIN` |
| 知识库 | `/api/knowledge/**` | 生成流程内部调用；调试和配置为 `PLATFORM_ADMIN` |
| 审计 | `/api/audit/**` | `AUDIT_ADMIN`、`PLATFORM_ADMIN` |
| 系统管理 | `/api/system/configs`、`/api/system/data-sources/**`、`/api/system/metrics/**` | `PLATFORM_ADMIN`、部分只读给 `AUDIT_ADMIN` |

### 16.3 数据隔离规则

- 普通用户默认只能访问自己创建或被分享授权的项目与报告。
- 管理员可访问全量数据。
- 文件访问必须先经过后端权限校验，再生成临时 URL。
- 下载、分享、删除、生成、外部 Agent 调用、知识库调用必须记录审计日志。

## 17. 幂等与并发规则

以下接口建议支持幂等：

| 接口 | 幂等依据 | 说明 |
| --- | --- | --- |
| `POST /api/projects/{projectId}/submit-generate` | `Idempotency-Key` + `projectId` + `userId` | 避免重复提交生成任务 |
| `POST /api/tasks/{taskId}/retry` | `Idempotency-Key` + `taskId` | 避免重复重试 |
| `POST /api/reports/{reportId}/regenerate` | `Idempotency-Key` + `reportId` | 避免重复创建版本 |
| `POST /api/files` | 文件 hash + 业务类型 + 用户 | 可选，避免重复上传 |

并发控制规则：

- 同一任务执行必须使用 Redis 分布式锁。
- Worker 消费任务前必须校验 MySQL 中任务状态。
- 任务最终状态以 MySQL 为准，Redis 队列只作为触发和加速机制。
- 报告生成过程中的阶段日志应按阶段追加，不在 Controller 中长时间阻塞。
